CUSTOM rate files¶

Custom rate files use the a similar syntax as reactions, with the following changes:

  • Only $ is evaluated

  • Automated evaluation of $ only available for vibrational levels

  • The card is name is used as the database handle

  • The subcards are of the form * TYPE ID. Here, “TYPE” defines one of the following reaction rate types:
    • RATE: a one-dimensional polynomial fit, for the EIRENE data

      • 9-polynomial fit in rising order

    • COEFFICIENT: a decay coefficient in s**-1, similar to the Einstein A coefficient

      • A single float

    • SIGMA: a fit for the reaction cross-section, which is integrated over a Maxwellian to yield the cross-section. Fit taken from K. Sawada and T. Fujimoto, “Effective ionization and dissociation rate coefficients of molecular hydrogen in plasma,” Journal of Applied Physics, vol. 78, no. 5, pp. 2913–2924, 1995, doi: 10.1063/1.360037.

      • Eth, q0, A, Omega, W, gamma, nu: as in the paper appendix

Just as for reactions, the line immediately after the subcard must contain the reaction definition, using the same syntax. The following line must then contain the appropriate reaction coefficients for the specified reaction type. If $ is utilized, the line after the reaction definition must contain a “v”, followed by the rate coefficients in the appropriate form on the following line. The “v”-coefficient pair must be repeated on the following lines vmax in order for all the required rates to be included in the model.
